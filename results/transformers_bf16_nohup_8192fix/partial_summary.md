## 关键结论

1. 单并发阶段可完成，但吞吐上限不高。`saturation_512_256` 在 c=1 时平均约 27.95 output tok/s，P95 延迟约 9.30s。
2. 并发升高后没有获得有效吞吐扩展，反而快速拉高延迟。`saturation_512_256` 到 c=4 时平均仅约 15.05 output tok/s，P95 延迟约 68.63s。
3. `chat_512_256` 在 c=32 时平均约 12.82 output tok/s，P95 延迟约 641.33s，说明该 Transformers FastAPI + model.generate baseline 不适合作为高并发推理服务。
4. 本次已完成结果足以说明该框架的高并发瓶颈，后续建议优先跑 vLLM、LMDeploy、SGLang 等具备 continuous batching / paged KV cache / 专用调度器的推理框架。
